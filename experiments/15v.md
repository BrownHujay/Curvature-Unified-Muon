# Series 15: Universal Muon — Kill the AdamW Crutch

**Files:** `cum/tensor/universal_muon.py`
**Status:** PENDING

## Motivation

Muon only orthogonalizes 2D hidden-layer weights (~60-70% of params). Everything else (embeddings, biases, LayerNorm) gets AdamW. This creates a Frankenstein optimizer where two parameter groups have completely different convergence dynamics sharing one training loop. Adam's slow adaptive warmup means embeddings/attention are at different optimization stages than hidden layers at any given step.

**Goal:** Single optimizer for ALL parameters. Uniform convergence behavior. No AdamW anywhere.

## Phase 1: Universal Muon (Simple)

Apply appropriate orthogonalization-like operation per parameter shape:

| Shape | Example | Operation |
|-------|---------|-----------|
| 2D (m×n), min(m,n)>1 | All weights, embeddings | NS orthogonalization (same as Muon) |
| 1D (d,) | Biases, LN scale/shift | Unit normalization: v/‖v‖ |

Key design decisions:
- **Embeddings get NS too.** They're 2D. No reason to exclude them.
- **1D gets normalized.** The "polar factor" of a vector is just its direction.
- **Single LR, single momentum** across all params.
- **Aspect ratio scaling** per param: sqrt(max(1, m/n)) for 2D, configurable for 1D.
- Start with basic Muon (no blending). Add combined/TD(λ) recipe in Phase 2 if Phase 1 shows promise.

## Key Hypothesis

Unified convergence dynamics > split optimizer, even if individual operations are suboptimal per group. The synchronization benefit (all params in same optimization phase) outweighs any loss from applying NS to embeddings or normalization to biases.

## Concerns
- Embeddings have different spectral structure (rare tokens → near-zero rows). NS might handle this poorly.
- LN scale params are initialized to 1.0 and barely move. Normalizing their gradient might over-amplify updates.
- Weight-tied head/embedding means the same param gets gradients from both LM head loss and embedding lookup — NS might struggle with this dual role.
- 1D scaling factor unclear — sqrt(d) from aspect ratio could be too aggressive.

## Tests

### 15a: Phase 1 Basic Universal Muon (A100 GPU, batch=32, 2000 steps)
| Config | Val Loss | vs Muon+AdamW | Time |
|--------|----------|---------------|------|
| Muon+AdamW (baseline) | 1.5216 | — | 15s |
| Universal all s1d=0.015 | 1.5280 | +0.0065 | 16s |
| Universal 2D + AdamW 1D | 1.5288 | +0.0073 | 14s |
| Universal all s1d=1.0 | 1.5449 | +0.0233 | 17s |

**Result:** NS on embeddings HURTS (+0.007). Embeddings are lookup tables, not linear transforms — orthogonalizing their gradients doesn't have the same geometric meaning. The Muon convention of excluding embeddings is structurally correct, not arbitrary.

### 15b: Per-Head Orthogonalization (A100 GPU, batch=32, 2000 steps)
| Config | Val Loss | vs Muon+AdamW | Time |
|--------|----------|---------------|------|
| **PerHead 4 slices (per head)** | **1.4991** | **-0.0253** | 19s |
| PerHead 3 slices (Q/K/V) | 1.5067 | -0.0178 | 16s |
| PerHead 12 slices (Q/K/V×head) | 1.5077 | -0.0167 | 33s |
| Muon+AdamW (baseline) | 1.5244 | — | 13s |

**Result:** Per-head orthogonalization gives -0.025 vs Muon. Sub-1.50 WITHOUT any blending. Bigger than the entire blending recipe (-0.018). Head separation > Q/K/V separation. 12 slices too fine (32×128 per slice gives NS too little to work with). ONE RUN — needs replication.

## Analysis

**15a (Universal Muon):** Failed as a unified optimizer. NS on embeddings hurts. The Muon+AdamW split is structurally correct — different parameter types genuinely need different optimization.

**15b (Per-Head Orthogonalization):** Major discovery. Standard Muon applies one NS call to the full (384, 128) QKV gradient, mixing all 4 heads' spectral profiles. PerHeadMuon splits into 4 slices of (96, 128) — one per head — and NS-orthogonalizes each independently.

Why this works (hypothesis):
1. **Different heads learn different features** with different spectral profiles. Mixing them in one NS forces shared equalization, which is suboptimal for each individual head.
2. **96×128 is still large enough** for NS to work well. 32×128 (12 slices) is too small — NS needs enough spectral diversity.
3. **Q+K+V within a head should stay together** — they interact tightly in attention computation. Separating Q from K from V (3 slices) is less useful than separating heads (4 slices).

This is **orthogonal to the blending work**. Per-head NS changes WHAT gets orthogonalized. Blending changes HOW the orthogonalization oscillation is managed. They should stack.

**Potential combined recipe:** PerHead 4 slices + TD(λ=0.5) + temporal + d=-1.0. If the effects are additive: -0.025 (per-head) + -0.009 (blending vs plain combined) = -0.034 vs Muon.

## Verdict
15a: FAILED — embeddings need Adam, not NS. Split is structurally correct.
15b: **MAJOR SUCCESS** — per-head orthogonalization gives -0.025 vs Muon (1 run). Needs replication. Potentially combinable with blending for even larger gains.
