# CUM 5v6: SVD + Custom SV Mapping

## Status: BEST RESULT (beats v5 and Muon)

## Core Idea
Since we proved (5v5) that NS ≈ scalar SV polynomial, we can replicate v5's multi-resolution blend as a pure SVD operation. The SVD gives EXACT singular values, and we apply the NS polynomial as a scalar function.

### ns_blend mode
Replicates v5's blend: `f(σ) = 0.85 * NS₅(σ) + 0.15 * NS₂(σ)`
But with SVD's exact SVs instead of matrix-level NS approximation.

### tilt mode
Novel: `f(σ) = NS₅(σ) * (1 + ε * (σ_norm - mean(σ_norm)))`
Adds a linear tilt to preserve SV ordering.

### Summary
- **Best config: save@2, blend=0.25** (marginally better than 0.15)
- save@2 >> save@1 (NS₁ too noisy, NS₂ is the denoised curvature sweet spot)
- Average improvement vs Muon: ~0.013 (better than v5's ~0.008-0.011)

## Key Insight
SVD-based SV manipulation is MORE PRECISE than matrix-level NS iteration. By computing exact SVs and applying the polynomial cleanly, we avoid floating point error accumulation across 5 NS iterations. This precision advantage is especially important for the blend operation, where small SV differences between NS₂ and NS₅ carry useful curvature information.

## Why tilt fails
The NS₂ intermediate contains SPECIFIC non-linear curvature information, not just "SV ordering." A generic linear tilt `ε * (σ - mean)` doesn't capture the complex structure of the NS₂ polynomial mapping. The blend works because it uses the EXACT NS₂ mapping.
