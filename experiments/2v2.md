# Experiment 2v2: SVD-Based Orthogonalization ⭐ (Key Discovery)

**File:** `cum/cum_2v2.py`
**Status:** FAILED (but produced critical insight)

## Hypothesis
Replace NS with exact SVD polar factor. Tests whether NS's approximation error is helping or hurting. Also test partial SV preservation (α > 0).

## Architecture
Momentum → SVD → Polar Factor (U @ Vh) → Optional SV preservation → Update

## Key Formula
```
U, S, Vh = torch.linalg.svd(u, full_matrices=False)
if α > 0:
    S_mod = S^α / mean(S^α)
    orth = U * S_mod @ Vh
else:
    orth = U @ Vh  # exact polar factor
```

## Cost
Full SVD per step — ~40% slower than NS₅ (601s vs 426s for 2000 steps).

## Critical Discovery
- **Exact polar factor is WORSE than NS's approximation by 0.0117**
- NS₅ SVs converge to ~0.877, not exactly 1.0
- This sub-unity convergence provides **implicit regularization** — slightly shrinking updates prevents overshoot
- SVD starts faster (better early orthogonalization) but NS's regularization wins in later training
- SVD is also ~40% slower

## Verdict
FAILED as optimizer, SUCCEEDED as insight. NS₅ > exact SVD. The iteration's convergence to ~0.877 fixed point is optimal. Don't try to "improve" NS by making it more accurate.
