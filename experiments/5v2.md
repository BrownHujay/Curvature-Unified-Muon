# CUM 5v2: Tunable Soft Equalization via SVD

## Status: FAILED

## Core Idea
NS maps ALL singular values to ~0.877 (hard equalization). But maybe full equalization is too aggressive. The optimal spectral filter might be BETWEEN identity and full equalization.

**Sigmoid SV mapping:**
```
σ → tanh(β * σ/σ_max) / tanh(β)
```

**Properties:**
- β → 0: identity (no processing, like SGD)
- β ≈ 3: moderate compression (preserves relative SV ordering)
- β ≈ 7: strong equalization (near-flat)
- β → ∞: hard equalization (all SVs → 1, exact polar factor)

Unlike NS, this mapping is:
- Continuously tunable (β is a real number, not integer NS steps)
- Monotonic (preserves SV ordering exactly)
- Smooth (no polynomial oscillation near the fixed point)

## Cost
- Full SVD per step (~40% slower than NS at our scale)
- No extra memory (just standard momentum buffer)

## Configs Testing
- β=3: moderate compression, preserves most SV structure
- β=7: strong compression, close to NS behavior

## Source
Novel — combines insight from deep research (NS ≈ Schatten-32 descent, 0.877 = optimal shrinkage) with tunable sigmoid filter design.

## Results
- **β=3**: val_loss = 1.6375 (+0.12 vs Muon)
- **β=7**: val_loss = 1.5875 (+0.07 vs Muon)

Higher β = more equalization = closer to Muon. Confirms full SV equalization is essential.

## Analysis
The tanh mapping fails because it doesn't lift small SVs enough. With β=7, σ=0.01 maps to only ~0.07 (vs NS mapping everything to 0.877). The tanh function is "soft clipping" — it compresses the top but barely affects the bottom. NS's fixed-point iteration fundamentally differs: it maps ALL SVs to the same target regardless of starting value.

**Key insight**: The correct mathematical framework for SV equalization is the power function σ^{1/(p-1)} from Schatten-p steepest descent, which lifts small SVs much more aggressively (σ=0.01 → 0.52 at p=8). This led to 5v3.
