# CUM 5v4: Adaptive Schatten-p (Per-Layer Condition-Based)

## Status: TESTING

## Core Idea
Extension of 5v3: instead of fixed p for all layers, adapt p based on the gradient momentum's condition number (σ_max/σ_min).

Intuition: layers with high condition number (spread SVs) need aggressive equalization (high p). Layers with low condition number (already equalized) can use less equalization (lower p), preserving useful spectral structure.

```
p_layer = p_min + (p_max - p_min) * sigmoid(k * (log_cond - log_cond_target))
```

Default config: p_min=4, p_max=64, cond_target=10, cond_steepness=2

## Cost
- Full SVD per step (~40% slower than NS)
- Negligible overhead for condition number computation (already have SVs)

## Source
Novel — combines Schatten-p framework from 5v3 with adaptive per-layer selection

## Results
- val_loss = 1.5309 (+0.016 vs Muon)
- Essentially tied with fixed p=32 (1.5308)

## Analysis
The condition-based adaptation doesn't help. Likely because most layers end up at high p values anyway (gradients typically have high condition numbers). The adaptation mechanism adds complexity without improving the SV mapping. Fixed p=6-8 is better.
