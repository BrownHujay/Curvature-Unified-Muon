## Hypothesis
If NS iterate blending works because it combines two "views" of the gradient at different processing stages, the principle should generalize. Test: two momentum buffers at different time constants (β=0.80 for curvature/recent, β=0.95 for direction/stable), blended before or after NS.

## Architecture
```
grad → [fast momentum (β=0.80)] → ┐
       [slow momentum (β=0.95)] → ┤→ blend → NS → update  (pre_ns)
                                   └→ NS each → blend → update  (post_ns)
```

## Key Formula
- **Pre-NS:** `u = (1-b)*nesterov(slow) + b*scale(nesterov(fast))`  then NS(u)
- **Post-NS:** `(1-b)*NS(nesterov(slow)) + b*scale(NS(nesterov(fast)))`

## Cost
- Pre-NS: One extra momentum buffer (same NS cost as Muon)
- Post-NS: Two NS passes (2x NS cost) + extra momentum buffer
