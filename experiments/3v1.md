# Experiment 3v1: Warm-Started Newton-Schulz

**File:** `cum/cum_3v1.py`
**Status:** PENDING

## Hypothesis
The polar factor changes slowly between optimizer steps. Instead of computing NS from scratch (5 steps), use the PREVIOUS step's NS output to warm-start and run fewer steps. This produces a fundamentally different trajectory through SV space.

## Architecture
Momentum → Blend(prev_orth, new_u) → NS(3 steps from warm start) → Update

## Key Formula
```
X₀ = (1-η) * prev_orth + η * (u / ||u||)
X₀ = X₀ / ||X₀||
orth = NS(X₀, steps=3)  # fewer steps needed from warm start
```

## Why This Is Different
- Muon: NS starts from scratch every step (cold). SVs go from arbitrary → 0.877.
- 3v1: NS starts from nearly-orthogonal matrix. SVs go from ~0.877 → different landing point.
- The NS trajectory visits completely different intermediate SV configurations.
- Also faster: 3 steps instead of 5 (saves 2 matmul rounds per weight).

## Cost
Same memory as Muon (one extra clone for prev_orth). 40% fewer matmuls (3 vs 5 NS steps after warmup).

## Analysis
- The warm start is too "sticky" — 50% previous polar factor prevents adaptation
- Starting NS from a nearly-orthogonal matrix means it's near the (unstable) fixed point at 0.868
- The iteration oscillates rather than converging to a useful direction
- 3 NS steps from near-orthogonal is basically a no-op — NS has nowhere to go
- Fundamental flaw: NS needs to start from a RAW gradient to produce meaningful trajectory

## Verdict
FAILED. NS must start from the raw gradient direction. Warm-starting from the previous output traps it near the unstable fixed point.
