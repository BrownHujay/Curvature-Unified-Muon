# Experiment 11: CUM v11 — Second-Moment Grafting

**File:** `cum/cum_v11.py`
**Status:** FAILED

## Hypothesis
Use Adam's second-moment estimate to reweight NS output element-wise. NS gives direction, Adam knows per-element magnitudes. Graft Adam's magnitude onto NS's direction.

## Architecture
Momentum → Multi-Resolution NS → Blend → Adam second-moment scaling → Update

## Key Formula
```
v = β₂*v + (1-β₂)*g²
adam_scale = 1/(sqrt(v_corrected) + ε)
update = (1-graft_α)*ns_update + graft_α*norm(adam_scale)*ns_update
```

## Cost
One extra buffer per weight (second moment v).

## Results
| Config | Val Loss | vs Muon | vs v5 |
|--------|----------|---------|-------|
| v11 graft=0.3 | 1.5126 | -0.0064 | +0.0049 |

## Analysis
- Beats Muon by 0.0064 (the v5 blend component helps)
- But loses to v5 alone by 0.0049 — grafting HURTS
- Per-element Adam scaling adds noise to the orthogonalized update
- NS's uniform magnitude is a FEATURE — redistributing magnitude element-wise breaks "equal SV treatment"
- Second moment too noisy at this scale for useful magnitude info

## Verdict
FAILED. Post-NS element-wise reweighting damages NS's core property.
