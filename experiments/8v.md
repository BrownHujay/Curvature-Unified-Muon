## Hypothesis
The only proven strategy (40+ experiments) is NS intermediate iterate blending (v5/5v6). We systematically explore the blending space:
1. Different blend pairs beyond NS₂+NS₅ (e.g. NS₂+NS₈, NS₃+NS₇)
2. Extended NS past 5 steps (7, 8) — more oscillation to cancel
3. Three-point blends (NS₁+NS₃+NS₅) — more interpolation points
4. SV-space geometric blending — does the blend operation matter?
5. Temporal input blending — EMA of past NS outputs

## Architecture
```
grad → momentum → [matrix NS / SVD] → blend iterates → update
                    ↓
            save intermediates at specified steps
            or compute SV mappings at specified iteration counts
```

## Key Formulas
- **Two-point:** `(1-b) * NS_n + b * scale(NS_k)`
- **Three-point:** `(1-a-b) * NS_n + b * scale(NS_j) + a * scale(NS_i)`
- **Geometric SV:** `S_out = NS_n(σ)^(1-b) * NS_k(σ)^b`
- **Input blend:** `(1-α) * NS₅(u_t) + α * scale(EMA(NS₅(u_{<t})))`

## Cost
- Matrix two/three-point: Same as Muon (saves intermediates, no extra matmuls)
- SVD modes: ~5x slower (full SVD per step)
- Input blend: One extra matrix in state (EMA buffer)
